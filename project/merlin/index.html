Live intake ↗ Merlin ↗

A realtime voice agent that qualifies inbound prospects in the browser, then books the briefing or captures the brief.

Situation

Discovery meetings were being spent qualifying callers who could have been qualified up front.

Task

  • Handle an inbound prospect end to end — greet, qualify, book.
  • Add a mode that returns a structured brief instead of a meeting.

Result

Live and embedded on the studio's intake page. Two variants, one binary, rollback-capable deploy.

My Contribution

  • Treated the model as untrusted — it can call tools, but it doesn't decide whether the work happened.
  • A live call ended claiming a brief was sent when none existed. Moved the gate out of the skipped tool into the dispatcher.
  • Decoupled the caller form from the model, which looped on greetings and called the tool late or never.

Key Results

20 Tools callable mid-conversation

18 for sales, 8 for intake — one registry, one binary.

~400 tests Across ~10.5k lines

Three guards written from real call failures.

Under 900ms Speech-to-audio budget

Frames forward on arrival; nothing is buffered.
