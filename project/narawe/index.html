Document intelligence for professional-services firms — scan or upload client PDFs, OCR them, and search across everything with fused keyword and semantic retrieval.

Situation

Law offices, accountants and consultancies hold their client history in scanned paper. Once a document is an image, it is invisible to search — the knowledge exists but nobody can retrieve it without knowing which folder to open.

Task

  • Get paper into the system with as little friction as possible, including straight from a phone camera.
  • Make everything searchable afterwards — by wording people half-remember, and by meaning when they don't.
  • Keep every firm's documents strictly separated from every other firm's.

Result

A working prototype running the full path: camera scan or bulk upload → OCR → keyword extraction → embedding → hybrid search, plus an auto-synthesised portfolio summarising everything known about a client. Prepared for deploy; not yet in production use.

My Contribution

  • Consolidated three AI providers onto one. The project started split across Clarifai for OCR, OpenRouter for keywords and OpenAI for embeddings; it now runs entirely on Gemini, which removed two vendor relationships and three failure modes.
  • Chose to fuse keyword and semantic search rather than pick one. People search for the exact phrase on the invoice and for the vague idea of what a letter was about — reciprocal rank fusion serves both from a single query.
  • Treated 'the model returned garbage' as a different failure from 'the network failed'. Transport errors retry with backoff; a response that is empty, too short, or echoing the prompt back triggers a separate content-aware retry that keeps the best attempt rather than returning nothing.
  • Made the client portfolio fire-and-forget, so an AI summary can never fail a document upload. The document reaching 'ready' is what the user is waiting on; the summary catches up on its own.
  • Set up scanning from a phone into a laptop-hosted stack over Tailscale, so the camera path could be tested against real paper rather than fixtures.

Key Results

Hybrid search with RRF fusion Keyword and semantic, one query

Postgres full-text and pgvector cosine run as separate passes, then fuse by reciprocal rank — with highlighted snippets so people can see why a result matched.

Task-typed embeddings Documents and queries embedded differently

A stored document and a search query are not the same kind of text; embedding them with different task types measurably helps recall.

Markdown to PDF engine 583 lines, written from scratch

Recompiles OCR'd text into a real searchable PDF — block parsing, font-metric word wrap, proportional table columns, pagination. The original scan is always preserved alongside it.
